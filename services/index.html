Production engineering for AI-built apps.

Security hardening

AI tools optimise for "make it work", not "keep it safe". They'll happily commit STRIPE_API_KEY="sk_live_..." straight into your repo, write auth checks that pass tests but not hostile traffic, and pull in dependencies that haven't been audited since 2022.

We sweep all of it. Secrets get pulled out of the codebase and into a managed vault. Auth flows get tested against the actual attacks — CSRF, session fixation, IDOR — not just the happy path. Dependencies get scanned, pinned, and gated in CI. We work the OWASP Top 10 like a checklist, because nearly every breach you read about started with one of those ten things.

What you get
  • Secret rotation + vaulted env management
  • Authentication and session review
  • Dependency audit and SCA scanning in CI
  • OWASP Top 10 sweep with prioritised fixes
  • Pen-test ready remediation report
  • Rate limiting and abuse protection

Cloud infrastructure

Vibe coding tools deploy to whatever's quickest — usually a single instance with a default config, no backups, and a database wide open to the public internet. That works for ten users. It does not work for ten thousand.

We rebuild your infrastructure on AWS or GCP with everything declared in Terraform: VPCs, security groups, autoscaling, managed databases, automated daily backups, and a disaster-recovery plan we've actually tested. Your staging and production environments are identical, separated only by a config flag. If we lost the database tomorrow, the IaC could rebuild the whole stack inside an hour.

What you get
  • VPC design and network policy
  • Terraform or Pulumi IaC for everything
  • Managed RDS / Cloud SQL with encrypted backups
  • Autoscaling and load balancing
  • Tested disaster-recovery runbook
  • Identical staging / production environments

CI/CD pipelines

Right now, "deploy" probably means dragging a file into Vercel and hoping. That's fine until the day you push something that breaks production at 11pm and you don't know which commit did it.

We replace that with GitHub Actions. Every push runs your tests. Every PR gets a preview environment a non-engineer can click around. Every deploy goes through staging first with one-click rollback if it fails. Type checks, lint, secret scanning, and dependency audits run on every change. You stop being scared of merging.

What you get
  • GitHub Actions workflow tuned to your stack
  • Automated tests, type-check, lint gates
  • Per-PR preview environments
  • Safe staging → production deploys
  • One-click rollback
  • Secret scanning on every commit
Read: CI/CD for non-engineers

Observability & monitoring

Most vibe-coded apps have no idea anything's wrong until a user emails. We give you eyes.

Sentry catches every exception with full stack traces and the user session that triggered it. Structured logs go to a queryable store, not just stdout. Uptime checks ping your critical paths every minute. Alerts fire to a Slack channel — but only the ones that matter, because alert fatigue is its own outage. By the time we're done, you'll know about a problem before your users do, with the data you need to fix it before it happens again.

What you get
  • Sentry error tracking, fully configured
  • Structured logging via BetterStack or equivalent
  • Uptime monitoring on critical user paths
  • Tuned alerts with on-call routing
  • Performance dashboards
  • Alert runbooks for the team
Read: monitoring for vibe-coded apps

ISO 27001 compliance

You can't sell to a regulated buyer — NHS, financial services, anything in the public sector — without certification. ISO 27001 is the most asked-for.

The good news: most of the work is already done if you've got the security and infrastructure pieces above sorted. We map your existing controls to the standard's requirements, write the policies you're missing, build the evidence trail auditors actually want to see, and walk you through the audit itself. Stage 1, Stage 2, certificate. We don't bill by the hour for compliance theatre — we get you certified and out the door.

What you get
  • Gap analysis against ISO 27001 controls
  • Policy templates customised to your business
  • Evidence collection and retention
  • Auditor-ready documentation
  • Stage 1 and Stage 2 audit prep
  • SOC 2 readiness as a side-effect

Code review & refactoring

AI-generated code has a recognisable shape. Massive functions doing four jobs. Variables named data, data2, result. Try-catches that swallow every error including the ones you wanted to see. Logic that "works" but breaks the moment you touch it.

We rewrite the parts that matter — the auth code, the payment code, the data flow — into something a human can maintain. We don't refactor for the sake of it. We refactor the bits that will bite you the day you hire your first real engineer and they ask "wait, what is this?"

What you get
  • Senior-engineer code review with prioritised findings
  • Refactor of high-risk modules (auth, payments, data)
  • Test coverage on critical user paths
  • Type safety where it's missing
  • Architecture diagram of the new shape
  • Written handover doc for future engineers
